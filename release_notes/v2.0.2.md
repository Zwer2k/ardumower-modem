# Release Notes

## HTTP Command Processing Fixes
* Fixed `BLE:COMM CRC ERROR: kI...` errors caused by heap memory leftover bytes contaminating HTTP POST command bodies. `trimHttpRequestBody()` now cuts at the first `\n` instead of only trimming trailing line endings, discarding stale encrypted data that leaked from `_tempObject` buffers.
* Fixed HTTP 400 "empty body" on large uploads (e.g., map waypoints). Added `recoverRequestBody()` to handle the timing race where ESPAsyncWebServer calls `onRequest` before all body chunks arrive, causing the `CommandRequest` constructor to read an empty `_tempObject`.

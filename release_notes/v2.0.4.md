# Release Notes

## WebSocket Connection Reliability
* **Full history on connect**: New WebSocket clients now receive the complete modem log ringbuffer (up to 300 lines) and terminal history (up to 200 lines) immediately upon connection, before the live stream continues
* **Dead-connection detection (ESP restart fix)**: Added a liveness check that closes the socket if no message has been received for 45 seconds. This ensures automatic reconnect after an ESP reboot, where the browser would otherwise keep a "zombie" socket open for minutes
* **Stale handler protection**: `open`/`close`/`error` event handlers now verify that the socket instance is still the current one before mutating the store state, preventing race conditions during rapid reconnects
* **Backend stability**: Fixed memory leak when a client reconnects with a reused WebSocket client ID; added `WS_EVT_ERROR` cleanup to correctly reset GPS/sensor ref counts

## Log & Terminal Export
* **Log dashboard**: The "Export CSV" backend link is now a "Download log" button that generates the CSV entirely from the client-side log array (no backend request)
* **Terminal**: Added a "Download Log" button that exports the visible terminal output as a `.txt` file

## OTA / Firmware Upload
* **STM32 flashing**: Added retry logic (3 attempts per block) for `sendCommand`, `sendAddress`, and `sendDataBlock` during STM32 OTA firmware upload
* **Mower upload**: Simplified response handling; removed unconditional automatic restart after upload

## Settings Save & Restart
* Replaced the `Ticker`-based delayed restart with a simple timestamp-based approach (`_restartAt` in `loop()`), fixing unreliable restarts after saving or resetting modem settings
* Settings response is now sent synchronously before the delayed restart triggers

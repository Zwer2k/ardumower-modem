# Release Notes

## Dashboard & MiniMap
* **Redesigned main dashboard**: Modernized layout with status cards for battery, charge state, current job, and GPS fix; set as the default view on open
* **Interactive MiniMap**: Added a compact map widget to the dashboard showing the mower position, perimeter, and recent driven track
* **Compass/rotation control**: Map editor and MiniMap now support rotating the map view with a compass-style control for easier alignment
* **Live track overlay**: Driven track is now rendered on the map editor with configurable length

## Map Management
* **Persistent map storage**: Maps are stored in SPIFFS with an index (`/maps/index.json`), supporting save, load, rename, delete, and metadata (area, hash, CRC, rotation)
* **Separate default map from loaded map**: Selecting a map in the editor no longer writes the active/default flag to flash on every change
* **"Set as default" button**: New button in the map management toolbar to explicitly persist the currently loaded map as the default startup map
* **Default marker in dropdown**: The default map is marked with a â˜… in the map selection dropdown, while the dropdown selection reflects the currently loaded map
* **Map upload to mower**: Added upload of the current map from the editor to the Ardumower rover
* **Draw mode improvements**: Extended map editing with drag edge, split perimeter, add points, and other drawing helpers
* **Mow settings dialog**: Added a settings dialog for mowing pattern, width, angle, border laps, and more

## Map Calculation & Routing
* **Waypoint calculation**: New path-planner integration using Clipper2 for perimeter/exclusion offset and route generation
* **Improved calculation dialog**: Better progress feedback and UI during waypoint calculation
* **Map CRC calculation**: Fixed and improved CRC computation to match Sunray expectations
* **Already-mowed way tracking**: Visual indication of already-mowed waypoints

## OTA / Firmware Upload
* **STM32 flashing**: Added retry logic (3 attempts per block) for `sendCommand`, `sendAddress`, and `sendDataBlock` during STM32 OTA firmware upload
* **Mower OTA upload**: Improved upload speed, simplified response handling, and a smoother progress bar
* **Reboot options**: Added reboot options for modem, rover, and GPS from the UI

## WiFi & Networking
* **Static IP configuration**: Added UI and backend support for static WiFi IP, netmask, gateway, and DNS
* **DHCP/static toggle**: WiFi settings now allow choosing between DHCP and static IP
* **Improved WiFi reconnection**: More stable reconnect behavior after disconnects

## Terminal & Log
* **Search bar in log view**: Added a search/filter bar to the log dashboard
* **Terminal context menu**: Improved terminal right-click context menu
* **Log download**: "Export CSV" is now handled client-side; terminal output can be downloaded as `.txt`
* **Reduced terminal spam**: Fixed duplicate/spammed command output when sending single commands

## GPS Dashboard & Live Map
* GPS dashboard fixes and stability improvements
* LiveMap now only displays the mower on the world map
* Goto is disabled when no map is loaded

## Dependencies & Build
* Updated `ArduinoJson` to 7.4.3
* Updated `NimBLE-Arduino`
* Updated `MQTT`, `AsyncTCP`, and `ESPAsyncWebServer`
* Updated `espressif32` platform to 7.0.0
* Added `Clipper2Lib` for polygon operations
* Added compiler switches: `ENABLE_MAP`, `ENABLE_LIVE_MAP`, `ENABLE_GPS_DASHBOARD`
* Split firmware source into folder structure under `src/`

## Stability & Performance
* Fixed heap fragmentation issues
* Fixed memory leaks during long-running operation and reconnects
* Improved WebSocket stability and reconnection handling
* Improved overall connection stability (HTTP, MQTT, WebSocket)
* Improved map upload communication reliability
* Various compilation warnings resolved

## WebSocket Protocol
* Added `setActiveMap` request type
* Map list response now includes both `activeId` (default map) and `currentId` (currently loaded map)

# Release Notes v2.3.1

## Map Editor Improvements

### New / Improved

- **JSON map import**
  - New dialog to import maps from JSON via the mower adapter and WebSocket.
  - Backend domain types and WebSocket messages extended for JSON map import.
  - Imported maps flow through the same rename/dirty workflow as intercepted maps.

- **Map editing UX refinements**
  - Smaller waypoint/perimeter points (~1.5 mm) while keeping a larger touch hit-area for easier editing.
  - New edit category selector in the map edit toolbar.
  - Improved layer render order and hit-area tuning for map elements.
  - Open docking and waypoint lines are rendered consistently.
  - The currently selected edit item is reset when edit mode ends, preventing stale selections.

## Path Planner Refactoring

### New / Improved

- **Standalone `lib/pathplanner` library**
  - The path planner has been extracted from `src/domain/path_planner.cpp` into a reusable `lib/pathplanner` library (`pathplanner.cpp`, `pathplanner.h`, `clipper_adapter.h`).
  - A dedicated test project (`test_pathplanner/`) with `make` build support allows offline validation of route generation.
  - The library uses Clipper2 for polygon offset, intersection, and difference operations.

- **Robust exclusion handling and boundary routing**
  - Route connectors are now routed along exclusion edges and the outer perimeter so they never cross holes or leave the mapped area.
  - Added safety pass that snaps any segment back to the boundary if it would exit the perimeter.

- **Ring pattern coverage and smoothness**
  - Ring offsets are now generated in parallel at multiples of the line spacing, avoiding unmowed strips between widely separated rings.
  - Added configurable tuning parameters (in `Tune` namespace):
    - `exclusionMarginFactor = 1.0` — line center distance from exclusion borders in mower widths
    - `ringSpacingFactor = 0.9` — slight overlap between rings for full coverage
    - `smoothingGrowth = 0.25` and `smoothingMax = 1.0` — progressive smoothing of inner rings
  - Inner rings gradually forget perimeter details, producing straighter lines while preserving the exact configured border distance on the outer ring.

- **Route simplification**
  - Final route is simplified to remove near-duplicate points and backtracking segments.
  - Removes exact U-turns (`A -> B -> A`) and partial backtracks (`A -> B -> C` where C lies on segment A-B).
  - Simplification only accepts shortcuts that stay inside the perimeter, so no shortcuts cut across concave corners or exclusions.

### Fixed

- Route segments could leave the perimeter when ring/zigzag/border-lap stages were connected by direct lines.
- Ring connectors could cross exclusion holes after reducing the exclusion margin.
- Border-lap segments landing exactly on exclusion edges triggered false positives in distance-to-border checks.
- Wide unmowed white strips appeared between rings because the old single-shrinking-ring algorithm did not cover large areas uniformly.

### Tests

- Extended `test_exclusion.cpp` with distance-to-border and exclusion-intersection checks.
- Added test scenarios for center/edge exclusions, border laps, triangular exclusions, and the user "mower map".
- Tests validate that no route segment intersects exclusions and that the minimum distance to the border matches the configured value.
